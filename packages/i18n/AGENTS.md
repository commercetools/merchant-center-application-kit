# i18n

Subdirectory context — supplements the root `AGENTS.md`. Only includes what
an agent needs to know to work safely in this package specifically.

## Purpose

Provides compiled react-intl translation messages for the Merchant Center shell, merging translations from app-kit, ui-kit, and community-kit so the application-shell can load them per-locale at runtime.

## Key Context

This package has a codegen pipeline that produces the `compiled-data/` directory:

1. `pnpm extract-intl` (root) extracts messages from all `*messages.ts` files across the monorepo into `data/core.json`.
2. `pnpm compile-intl` (root) or `pnpm compile-data` (this package) compiles the JSON files in `data/` into pre-parsed AST JSON in `compiled-data/` using `@formatjs/cli`.

The `data/` directory contains source translation JSON (one file per locale). The `compiled-data/` directory contains the build output consumed at runtime via dynamic imports (webpack chunk splitting by locale).

## How To Work Here

- To add or update a translatable string, edit the relevant `*messages.ts` file in the consuming package, then run `pnpm extract-intl` from the root to regenerate `data/core.json`.
- After editing any file in `data/`, run `pnpm compile-intl` from the root to regenerate `compiled-data/`.
- Tests: `pnpm --filter @commercetools-frontend/i18n run test`

## Gotchas

- Files in `compiled-data/` are generated — do not hand-edit them. They are overwritten by `pnpm compile-intl`.
- `data/core.json` is also generated by `pnpm extract-intl` — manual edits will be lost. The other locale files in `data/` (e.g. `de.json`) come from the translation service and should not be hand-edited either.
- Adding a new supported locale requires adding a new dynamic import branch in `src/load-i18n.ts` (explicit switch cases are used for webpack chunk naming).
